## Функції ##
### dietician-features ###
1) Створений клас DieticianClient, який підтримує зв'язок між самим дієтологом
та клаєнтом, а також розрізняє профіль користувача та дієтолога
2) view_patient_stats - забезпечує можливість дієтологу переглядати профіль користувача, та всі його статистики імпортовані з інших модулів
3) get_my_clients Показує дієтологу всіх його клієнтів і він може обрати когось конкретного для продовження чату з ним
4) get_daily_summary функція видає по запиту дієтолога всю інформацію про клієнта за його конкретним id
5) verify_access — перевіряє, чи має дієтолог доступ до конкретного клієнта, підтверджуючи роль користувачів та наявність зв’язку між ними
6) assign_patient — дозволяє дієтологу призначати клієнтів, створюючи зв’язок між дієтологом і користувачем у системі
7) analyze_nutrition_logs — аналізує харчові записи клієнта за заданий період часу, підраховує сумарні показники БЖВ та калорій і порівнює їх із встановленими цілями харчування

### health_database ###
1) Створенно клас Profile який збирає інформацію про користувача та записує її в наші таблички
2) створено клас StressTracker він зберігає інформацію про рівень стресу який вказав користувач, а такод дозволяє залишити коментар, стосовного свого стану
3) Створено клас Nutrition створює окрему табличку де зберігаються данні про калорії, та кількість прийомів їжі, а також інформація про білки. жири, вуглеводи та клітковину
4) створено клас SleepRecord який так само окрумо записує інформацію про кількість сну, яку вказав користувач
5) Створено клас DieticianClient який створює окрему табличку для запам'ятовування профілів саме дієтологів
### stress-tracker ###
1) Створення класу StressRecord який валідує дані користувача
2) check_danger_threshold Перевіряє наскільки високий рівень стресу в користувача ( якщо стабільно високий цей користувач буде закріплений за дієтологом)
3) calculate_average_stress Обраховує середній стрес користувача
4) get_history Записує в дата бейс після всіх обробок та валідацій інформації від користувача
### sleep-tracker ###
1) SleepRecord — модель для збереження інформації про сон користувача, включає час засинання, пробудження та автоматично обчислює тривалість сну
2) get_avg_sleep — обчислює середню тривалість сну користувача за вказану кількість днів, аналізуючи всі записи сну за відповідний період
3) get_chart_data — формує дані для побудови графіка сну, повертаючи список дат та кількість годин сну користувача за останні 30 днів
### nutrition module ###
1) Nutrition — модель для збереження щоденних даних харчування користувача, включає кількість прийомів їжі, показники білків, жирів, вуглеводів, клітковини та калорій
2) calculate_calories — автоматично обчислює загальну кількість калорій на основі макронутрієнтів (білки, жири, вуглеводи та клітковина)
3) save — перевизначений метод збереження моделі, який перед записом у базу автоматично розраховує калорійність харчування
4) NutritionService.create_entry — сервісна функція для створення запису харчування користувача, виконує валідацію введених даних та додає новий запис у систему
5) NutritionValidator.validate — перевіряє коректність введених харчових даних перед створенням запису (діапазони значень, формат та логічність даних)
